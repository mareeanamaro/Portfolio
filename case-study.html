CASE STUDY - FLICKING THROUGH FLICKS (REACT)

WHAT?

Flicking through Flicks is a web app created using the MERN stack. It provides information about movies, directors, and genres. The goal was to create a web app about movies that will allow users to discover information about these particular movies and curate a list of their favourites.

WHY?

This project came about as part of my Full Stack Web Development Course with Career Foundry. I wanted to build a full-stack app, from the database and API to the client side, to get an understanding of how all the parts connect and work together.

HOW?

I worked on this project in the evenings and on weekends, over the span of three months. I used Express, Mongoose, BCrypt (Server-Side) and Nodejs, React, React-DOM, Parcel, Axios, Redux, Bootstrap (Client-Side).

WHO?

I was the lead developer on the project and would like to thank Martin Macharia and Ivan Bila, whose help was invaluable.

WHERE?

Live App | Client-Side Code | API

The Process

STEP 1 - BUILD THE API & BUSINESS LOGIC LAYER

I designed what the app would do, and the operations the API would support. I created a RESTful API using Node.js and Express. The business logic layer was written with with Mongoose. I used BCrypt for password encryption and Passport for authentication. For the testing I used Postman.

STEP 2 - BUILD THE DATABASE

I built a non-relational database of movies and users using MongoDB. I used a non-relational database because it is more flexible and the app is simple enough to accommodate this type of database.

STEP 3 - BUILD THE CLIENT-SIDE IN REACT

I built the client-side using React and React-Redux. The app client-side include Login and Registration views for unauthorized users, a Main view which shows all movies, Movie-specific views and a Profile view, which allows the users to edit their details and list of favorite movies.

This was my first time working with React and React-Redux and I felt the steep learning curve, but it was ultimately very rewarding to get it all done and working. I learned a lot about how current development trends are so reliant on packages and that you may have to downgrade the tools you are using to get everything working in tandem.

This build and the calls I had with my mentor Martin to help me when I was stuck really developed my debugging skills and my understanding of single page applications and user interfaces.

Ultimately I was able to complete my objective. This project really taught me how important collaboration is when it comes to big projects and that often it is better to stop and take a moment to re-learn concepts so your end product can be as solid as possible. I look forward to building more projects in the MERN stack and honing my skills.
